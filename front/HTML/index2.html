html의 역사

(1990년대 ~ 2000년대 초반) 탄생 (1989~1991): 팀 버너스-리(Tim Berners-Lee)가 CERN에서 연구 문서 공유를 위해 HTML을 고안. HTML 1.0 (1993): 기본적인 텍스트 포맷과 하이퍼링크만 지원. HTML 2.0 (1995): 폼 요소와 테이블 지원, 최초의 표준화된 버전. HTML 3.2 (1997): 스크립트와 스타일시트 지원, 멀티미디어 요소 확장. HTML 4.0 (1997): DOM(Document Object Model) 도입, 접근성 개선, CSS와의 통합 강화. 특징: 웹은 주로 텍스트와 단순 이미지 중심. 브라우저마다 해석 방식이 달라 호환성 문제가 많음. 표준화 필요성이 대두되며 W3C(World Wide Web Consortium)가 중심 역할 수행.

XHTML (2000): XML 기반으로 더 엄격한 문법을 요구했지만, 개발자들에게 부담이 커서 널리 확산되진 못함. HTML5 (2014 공식 권고): 비디오·오디오 태그로 멀티미디어 직접 지원. canvas 태그로 그래픽 처리 가능. 모바일·반응형 웹을 고려한 설계. 시맨틱 태그(article, section, header 등)로 문서 구조 명확화. 현재 (2020년대): HTML은 단순한 마크업 언어를 넘어 웹 애플리케이션 플랫폼으로 자리잡음. 접근성, 성능, 보안, 모바일 친화성 중심으로 발전. W3C와 WHATWG가 협력해 “Living Standard” 방식으로 지속 업데이트.

html의 활용

기본 활용 (문서 구조화) 웹 페이지의 뼈대를 만드는 용도 제목, 문단, 리스트, 링크, 이미지 등을 구조적으로 표현

확장 활용 (웹 애플리케이션 기반) CSS, JavaScript와 결합해 동적인 웹 환경을 구축 반응형 디자인, 인터랙션, 멀티미디어 삽입, 폼 처리 등

html 사용시 주의점

기본적인 주의 사항 태그 닫기 : 태그를 열었다면 동일한 태그로 닫아야 한다. 중첩 규칙 지키기 : 태그는 올바른 순서로 열고 닫아야 한다. 시맨틱 태그 사용 : HTML 속성 값은 큰 따옴표로 감싸는 게 안전하다. 대소문자 일관성 : HTML5는 대소문자 구분을 안 하지만, 소문자로 쓰는 게 관례다.

심화적인 주의 사항. 접근성 고려 : 이미지에는 반드시 alt 속성을 넣어야 시각장애인용 스크린디어가 읽을 수 있다. 구조와 디자인 분리 : HTML은 구조, CSS는 디자인, JS는 동작을 담당하도록 분리해야 유지보수가 쉽다. 중복 콘텐츠 피하기 : 같은 내용을 여러 태그로 반복하면 SEO에 불리하다. 표준 준수 :W3C표준을 지켜야 브라우저 호환성이 좋아집니다. 불필요한 태그 남용 금지 : br 태그나 div 태그를 과도하게 쓰지 말고, 의미있는 태그로 구조화해야 한다.
